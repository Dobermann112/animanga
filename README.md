# AniManga

## ■ 概要

AniMangaは、ユーザーが「漫画・アニメ作品」をレビュー形式で投稿し、他ユーザーと共有できるレビューSNSです。

作品単位での投稿に特化し、ユーザーがおすすめしたい作品や感想を簡潔に発信・発見できる体験を提供します。

また、本リポジトリは、Next.jsを中心としたフルスタック開発の学習を目的として開発しています。実務を意識した設計・実装・PR運用を取り入れながら、段階的に機能追加を行っています。

---

## ■ コンセプト

* 漫画・アニメに特化したレビューSNS
* 「作品 × 体験」を共有するプラットフォーム
* 投稿のハードルを下げつつ、質の高い情報共有を実現
* 学習を兼ねた実践的なフルスタック開発

---

## ■ 主な機能（実装済み）

### 認証機能

* ユーザー登録
* ログイン / ログアウト
* セッション管理（NextAuth）
* ログインユーザーに紐づく投稿作成
* 投稿者のみ編集・削除可能な認可制御

### 投稿機能

* 漫画・アニメ作品のレビュー投稿
* 概要（必須）
* 評価（★1〜5）（必須）
* 詳細レビュー（任意）
* レビュー対象（漫画 / アニメ / 両方）
* 投稿編集
* 投稿削除
* 投稿詳細表示

### タイムライン

* 投稿一覧表示（新着順）
* 人気順表示（いいね数ベース）
* 保存済み投稿表示
* カテゴリフィルター

  * すべて
  * 漫画
  * アニメ
* タイトル検索
* 検索・フィルター・並び替えの複合条件対応
* ページネーション

### ユーザーアクション

* いいね機能
* ブックマーク機能
* コメント投稿
* コメント削除
* いいね数表示
* ブックマーク数表示
* コメント数表示
* ユーザーフォロー機能

### マイページ

* ログインユーザー専用マイページ
* ユーザー情報表示
* プロフィール本文（BIO）表示
* 投稿数 / いいね数 / 保存数の表示
* 自分の投稿一覧
* いいねした投稿一覧
* 保存した投稿一覧
* フォローしているユーザー一覧
* マイページ内タブ切り替え
* マイページ内ページネーション
* プロフィール編集ページへの導線

### ユーザープロフィール

* ユーザー詳細ページ
* usernameを利用したプロフィールURL
* 他ユーザーのプロフィール閲覧
* 他ユーザーの投稿一覧表示
* プロフィール本文（BIO）表示
* 自分のプロフィールへアクセスした場合はマイページへリダイレクト
* 投稿一覧・投稿詳細・コメント欄からプロフィールページへ遷移

### プロフィール編集

* 表示名の編集
* プロフィール本文（BIO）の編集
* フロントエンド・API双方でのバリデーション
* Headerの表示名をDBの最新情報から表示

## ■ 検索・フィルター例

以下のような条件を組み合わせて一覧表示できます。

* 人気順 × 漫画
* 人気順 × アニメ
* 保存済み × 漫画
* 保存済み × アニメ
* タイトル検索
* タイトル検索 × ページネーション

URLクエリを利用して一覧状態を管理しています。

例：

```txt
/?sort=popular&target=MANGA&q=ワンピース&page=2
```

## ■ 学習目的

本アプリは、Next.jsを用いたフルスタック開発を実践的に学習することを目的としています。

主な学習テーマは以下の通りです。

* Next.js（App Router）の理解
* Reactの状態管理・コンポーネント設計
* Server Component / Client Component の使い分け
* SSR / CSRの使い分け
* Route HandlersによるAPI設計
* Prismaを利用したデータ設計
* Prisma Migrationの理解
* NextAuthによる認証・認可
* URLクエリを利用した検索・フィルター・ページネーション設計
* usernameを利用した動的ルーティング設計
* フォーム処理とプロフィール編集
* フロントエンド・API双方でのバリデーション設計
* SessionとDBの責務分離
* TypeScriptによる型安全な実装
* 実務を意識したPRベースの開発フロー
